## Детализация требований

### Дневник мыслей (таблица КПТ)
#### Описание
Пользователь заполняет структурированную таблицу для работы с тревожными мыслями по методике КПТ: ситуация -> автоматическая мысль -> эмоция -> переформулировка

#### Функциональные требования (ФТ)
1. Пользователь может создать новую запись в дневнике мыслей.
2. Пользователь может заполнить 4 обязательных поля: «Ситуация», «Мысль», «Эмоция», «Переформулировка».
3. Пользователь может выбрать эмоцию из выпадающего списка: тревога, страх, стыд, злость, грусть, раздражение, вина.
4. Пользователь может оценить интенсивность эмоции по шкале от 1 до 10 (ползунок или цифровой ввод).
5. Пользователь может сохранить запись (кнопка «Сохранить»).
6. Пользователь может редактировать ранее сохранённую запись.
7. Пользователь может удалить запись (с подтверждением «Вы уверены?»).
8. Пользователь может просмотреть все свои записи в виде хронологического списка с датой и кратким превью (первые 50 символов из «Ситуации»).
9. Пользователь может открыть полную версию любой записи по тапу.
10. Пользователь может получить подсказку-пример для каждого поля (иконка «?» рядом с полем).
11. Пользователь может отфильтровать записи по дате (сегодня, эта неделя, этот месяц, всё время).
12. Пользователь может заполнять поля в любом порядке (без блокировки последовательности).

#### Нефункциональные требования (НФТ)
1. Приложение должно автоматически сохранять черновик каждые 10 секунд при наличии изменений.
2. Время открытия экрана дневника мыслей после нажатия на иконку не более 1 секунды.
3. Приложение должно поддерживать офлайн-режим: заполнение без интернета с синхронизацией при восстановлении соединения.
4. Данные пользователя (записи дневника) должны храниться в зашифрованном виде.
5. Интерфейс должен быть доступен для людей с дальтонизмом (цветовая индикация эмоций + текстовые метки).
6. Приложение должно корректно работать на экранах от 5" до 7" (адаптивная вёрстка).
7. При переполнении текста в поле (более 1000 символов) должна появляться прокрутка, а не обрезка текста.
8. Приложение не должно терять данные при случайном закрытии (swipe up или вылете).

### Анимация получения XP после экспозиции
#### Описание
После выполнения пользователем экспозиции (упражнения на выход из зоны комфорта) приложение показывает анимацию получения опыта (XP) с увеличением уровня персонажа

#### Функциональные требования (ФТ)
1. После успешного завершения экспозиции автоматически запускается анимация получения XP.
2. Анимация показывает количество полученного опыта (например, «+15 XP»).
3. Анимация показывает текущий уровень персонажа после начисления XP.
4. Анимация показывает шкалу прогресса до следующего уровня (% заполнения).
5. Пользователь может пропустить анимацию одним тапом по экрану.
6. Длительность анимации не более 2 секунд (если не пропущена).
7. При повышении уровня дополнительно показывается анимация «LEVEL UP!» с визуальным эффектом (вспышка/свечение).
8. Звуковое сопровождение анимации можно отключить в настройках приложения.
9. При повторном просмотре (например, из истории) анимация не запускается повторно.
10. Количество XP за экспозицию зависит от сложности: базовая экспозиция = 10 XP, продвинутая = 20 XP.
11. Пользователь видит суммарное количество XP на экране профиля.

#### Нефункциональные требования (НФТ)
1. Анимация должна работать плавно с частотой 60 fps на устройствах от iPhone X / Android 10.
2. Анимация не должна содержать быстро мигающих элементов (риск эпилепсии).
3. Анимация должна работать без интернета (локально, без запросов к серверу).
4. Потребление батареи во время анимации не более 1% за 100 показов.
5. Анимация должна быть адаптивна для экранов от 5" до 7" (элементы не обрезаются).
6. При низкой производительности устройства (старые модели) анимация автоматически упрощается (отключаются частицы, остаётся текст).
7. Анимация не должна блокировать основной поток UI (использовать асинхронную отрисовку).
